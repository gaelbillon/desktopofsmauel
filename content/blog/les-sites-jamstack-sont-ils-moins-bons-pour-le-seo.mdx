Il est vrai qu'il y a quelques années, un site qui utilisait intensément le Client Side Rendering comme une PWA ReactJS par exemple, pouvait poser des problèmes pour le référencement. C'est parce que la plupart du temps, avec un site JAMstack, les navigateurs (et les robots de moteurs de recherche) vont d'abord télécharger un document HTML mais ce document est quasiment vide. Ensuite le code Javascript est téléchargé. Ce code Javascript va ensuite permettre de remplir le contenu de la page. Donc la première chose que voient les crawlers des moteurs de recherche est une page vide. Avec le server side rendering, comme avec NextJS ou Wordpress, le navigateurs téléchargent directement une page HTML avec tout son contenu. Il était peut être vrai il y'a quelques années que les robots avaient du mal avec ça. Mais ce n'est plus un problème aujourd'hui. Et les générateurs de sites statiques sont dotés de nombreux plugins pour le SEO, les données structurées, etc. Le chargement très rapide des pages d'un site statique offre un gros bonus SEO chez la plupart des moteurs de recherche. C'est devenu un élément déterminant du classement dans les pages de résultat.
